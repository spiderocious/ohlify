Observability map

What to check first, in order, when something looks off.

  1. The call row. SELECT * FROM calls WHERE id = .... Status, timestamps, related_booking_id, related_journal_id.
  2. The call_events table. Every state transition is recorded here. Read top-down to see the lifecycle.
  3. Wallet journals related to the call. SELECT * FROM journal_entries WHERE related_call_id = '...'.
  4. Outbox events. SELECT * FROM outbox WHERE aggregate_id = '<call_id>'. Were the side-effects fired?
  5. Server logs. Search for the call id. Pino logs are JSON; jq filters.
  6. Agora dashboard. Channel logs for the matching channel name.

Common scenarios

Caller says "I joined but no one was there"

To fill in. Check: callee join event in call_events, Agora channel logs.

Callee says "the call ended on its own"

To fill in. Token expiration? Network drop? Server-side hangup?

Wallet says they were charged but call never happened

To fill in. Reservation posted but no settlement. Refund eligible.

"Call shows completed but money never moved to my wallet"

To fill in. Settlement journal posted? Outbox event published? Withdrawable balance refresh.

Reconciliation worker fired ERROR (drift detected)

Hard stop. Halt all wallet writes. Don't approve refunds, don't process withdrawals, don't run admin journal endpoints. Investigate before anything else moves. Likely cause: someone bypassed the journal API and ran a raw UPDATE on account_balances.

Agora bill spike

To fill in during phase 1. Check Agora dashboard usage chart by channel; identify the runaway. Rotate App Certificate if compromised.

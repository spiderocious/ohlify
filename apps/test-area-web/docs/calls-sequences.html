Scenarios to cover

  1. Happy call. Booked, paid, both join, both leave, settled.
  2. Caller no-show. Callee waits, no-show window expires, full refund.
  3. Callee no-show. Caller waits, full refund, professional gets a strike.
  4. Both no-show. Money back, no strike for callee.
  5. Mid-call disconnect — caller side. Pro-rate, settle the partial.
  6. Mid-call disconnect — callee side. Same logic, mirrored.
  7. Cancellation outside window. Refund per cancellation policy.
  8. Cancellation inside window. Partial refund or none, per policy.
  9. Network flap during call. Re-join window allowed; settle includes the gap.
  10. Pro-rate edge: 0 minutes connected. Treat as no-show.

Each scenario will include

  • Sequence diagram (mobile · backend · Agora · wallet · outbox).
  • Database state at each step (booking row, call row, journal entries).
  • Outbox events fired and which emails/pushes go out.
  • Failure modes and what we do about them.
  • Where in the code each step lives — file paths and function names.

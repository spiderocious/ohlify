Agora · account & key setup

Phase 1 walkthrough. Backend integration is wired (token issuance + webhook handler); this page is what you do on the Agora dashboard side. Notes from the doc you'll revisit when something breaks.

1. Concepts (lock these before clicking around)

App ID
Public identifier for the Agora project. Mobile sees this. Goes in AGORA_APP_ID env var.
App Certificate
Secret used to sign tokens (HMAC-SHA256). Backend ONLY. Treat like a database password. Goes in AGORA_APP_CERTIFICATE.
Channel
A named room. Two clients with the same channel name + valid tokens can talk. We name channels call_<call_id>.
UID
32-bit unsigned int per user inside a channel. Our backend hashes the user id to derive a stable UID via agoraUidForUserId.
RTC token
Signed blob, time-limited (1 hour). Required to join in secured mode. Backend mints via issueAgoraRtcToken.
Role
Publisher (can send media) or audience (receive only). Both caller + callee are publishers.
AccessToken2 ("007")
The token format we mint. Implemented from scratch in lib/agora/token.ts; no Agora SDK install required.

2. Account creation walkthrough

  1. Sign up at agora.io. Use a work email if possible — billing is per-org.
  2. From the Agora Console, create a new Project. Name it something like ohlify-dev.
  3. Choose authentication mode: "Secured mode (App ID + App Certificate)". Never pick App-ID-only mode — that's prod-unsafe and won't work with our backend (we always sign tokens).
  4. The project page shows the App ID. Copy it. Paste into apps/backend/.env as AGORA_APP_ID=<...>.
  5. Find the App Certificate section. Click Show. Copy. Paste as AGORA_APP_CERTIFICATE=<...>. Never commit this. Never log this.
  6. Restart the backend so the new env vars are picked up. pnpm dev in apps/backend.

3. Services to enable

  • RTC. ✅ On (default). This is the audio/video.
  • RTM. ❌ Off. We don't use it — call lifecycle is driven by our backend's join/leave endpoints + Agora's notification webhooks.
  • Cloud Recording. ❌ Off. Deferred until reviews/disputes need it. Adds cost + legal complexity.

4. Webhooks (Agora "Notifications V2")

  1. In Agora Console, go to Project → Notifications. Enable.
  2. Set the webhook URL to https://<your-backend>/api/v1/webhooks/agora. For local dev, use a tunnel (ngrok / cloudflared).
  3. Generate a webhook secret. Copy. Paste into apps/backend/.env as AGORA_WEBHOOK_SECRET=<...>.
  4. Subscribe to events: channel-create (101), channel-destroy (102), user-join (103), user-leave (104). Skip the rest — they're noisy.
  5. Without AGORA_WEBHOOK_SECRET set, the handler accepts unsigned events (dev convenience). Set it before staging.

5. Pricing snapshot (check current rates on the dashboard)

  • Free tier: 10,000 minutes/month across all media. Easily covers dev + early QA.
  • Pay-as-you-go after that, billed per "channel-minute" per user:
    • Audio: ~$0.99 / 1000 min
    • HD video: ~$3.99 / 1000 min
    • Full HD: ~$8.99 / 1000 min
    (Numbers from current published rates — verify against your account before launch.)
  • Two-party call cost: A 30-min HD video call between caller + callee = 2 × 30 = 60 channel-minutes ≈ $0.24 in raw Agora cost. Our 15% platform fee on a ₦400 (~$0.30 with 1300 NGN/USD) call has zero margin if we go HD video; ₦1000+ rates work.

6. What's already wired in our backend

  • lib/agora/token.ts — RTC AccessToken2 implementation. Mints publisher tokens scoped to (channel, uid, expires).
  • lib/agora/webhook.ts — HMAC verifier + event-type constants.
  • Env: AGORA_APP_ID (required), AGORA_APP_CERTIFICATE (required), AGORA_WEBHOOK_SECRET (optional in dev).
  • Webhook route: POST /api/v1/webhooks/agora — wired in phase 4.
  • Token issuance from features happens via POST /api/v1/calls/:id/join — ships in phase 4.

7. Sandbox vs production

Plan: dev + staging share one Agora project (with a low spend cap), production gets a separate project with separate App ID + App Certificate. Channel naming (call_<call_id>) already namespaces by environment because call IDs are unique per-environment database.

8. Operational checklist

You should be able to do all of these from the Agora dashboard:

  1. Set a per-day spend cap so a runaway loop can't burn $1000.
  2. Rotate the App Certificate. (After rotation, restart backend — old in-flight tokens stay valid until expiry, ~1h.)
  3. See connected-minute usage by day, by channel, by user.
  4. Read channel logs for a specific channel name to debug a complaint.
  5. Generate test tokens from the dashboard "Token builder" tool to compare against ones our backend mints.
